Accounts

Accounts are necessary to track user progress. Please Sign up to be able to Sign in and get started! After signing in, you will be directed to the Homepage.

Content

Homepage


The homepage is where the magic happens. It contains all of the content. This primarily includes Units, but also features a menu on the left-hand side, your daily challenge on the right-hand side, and a language select.

Units


Each unit is a subject in your selected programming language. For example, the first unit in Java teaches you how to declare variables, which are the building blocks of code. Units are comprised of three Lessons and one Milestones.

Lessons


Lessons are ordered by difficulty level under each unit. Each lesson is a set of practice questions at its difficulty level. These questions will be one Multiple Choice, followed by a Drag & Drop, followed by a Short Response.

Milestones


Milestones are the final boss of a unit. There is only one per unit, as they represent the cumulative content of that unit. They are comprised of three Short Response questions of varying difficulty.

Questions

Multiple Choice


Multiple Choice questions are exactly what you expect. You are presented with a list of four answer choices to the given prompt, and only one of them will be correct. Try your best to select the correct choice and submit!

Drag & Drop


Drag & Drop questions are a bit trickier. You are presented with a list of code blocks that you must drag into the correct order according to the given prompt. Try your best to arrange the correct ordering and submit!

Short Response


Short Response questions are the hardest. You are presented with an empty text box to type in a correct answer to the given prompt. Try your best to write the desired code and submit!

Short Response Formatting

Short Response questions assume a certain whitespace style. In code, spacing between keywords is often flexible and is referred to as whitespace. However, our validator requires one space between keywords and operators (except for semi-colons).

The whitespace style in Multiple Choice questions follow these requirements exactly. It is easiest to simply mimick the style of these options when it comes time to do a Short Response question.

For example, the correct answer to a question might be int foo = 1;. The following answers might technically be correct, but they violate the whitespace style and thus our validator will mark them as incorrect.
  • int foo= 1;
  • int foo =1;
  • int foo=1;
  • int foo = 1 ;
